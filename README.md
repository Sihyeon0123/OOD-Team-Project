## 프로젝트 개요
본 프로젝트는 기존에 구축된 **자바 스프링 부트 기반의 James Web Mail Server**를 대상으로, 소프트웨어 유지보수의 네 가지 핵심 유형인 **교정**, **적응**, **완전화**, **예방 유지보수**를 적용하여 시스템을 개선하는 것을 목표로 합니다. 
사용자에게 더욱 편리하고 안정적인 웹 메일 서비스를 제공하기 위해, 전반적인 소프트웨어 아키텍처 구조와 동작 흐름에 대한 이해를 바탕으로 시스템의 안정성과 성능을 지속적으로 향상시켜 나갔습니다.


## 유지보수 유형
본 프로젝트에서는 소프트웨어 유지보수의 네 가지 유형을 모두 적용하여 James Web Mail Server의 품질을 향상시켰습니다.
1. **교정 유지보수 (Corrective Maintenance)**: 시스템에서 발견된 오류나 결함을 수정하는 활동입니다.
    - 목표: 시스템의 올바른 작동을 보장하고, 사용자 불편을 초래하는 버그를 제거합니다.
    - 예시: 메일 전송 실패 오류 수정, 첨부 파일 다운로드 문제 해결.
2. **적응 유지보수 (Adaptive Maintenance)**: 시스템이 변화하는 환경(하드웨어, 소프트웨어, 운영체제 등)에 적응할 수 있도록 수정하는 활동입니다.
    - 목표: 시스템이 새로운 환경에서도 안정적으로 작동하도록 합니다.
    - 예시: 최신 Java 버전 또는 Spring Boot 프레임워크 업데이트에 따른 코드 수정 및 호환성 확보.
3. **완전화 유지보수 (Perfective Maintenance)**: 시스템의 기능적 개선, 성능 최적화, 사용성 향상 등을 목표로 하는 활동입니다.
    - 목표: 사용자 경험을 개선하고 시스템의 효율성을 높입니다.
    - 예시: 메일 검색 기능 강화, UI/UX 개선, 불필요한 코드 제거를 통한 성능 최적화.
4. **예방 유지보수 (Preventive Maintenance)**: 잠재적인 오류를 사전에 식별하고 수정하여 미래에 발생할 수 있는 문제를 예방하는 활동입니다.
    - 목표: 시스템의 장기적인 안정성과 유지보수성을 확보합니다.
    - 예시: 코드 리팩토링, 보안 취약점 패치, 문서화 개선.

1. **요구사항 조사 (Requirements Analysis)**:
   - 기존 시스템의 문제점 및 사용자 요구사항을 파악하기 위해 심층적인 조사를 수행했습니다.
   - 사용자 피드백, 기존 버그 보고서, 성능 지표 등을 분석하여 유지보수 우선순위를 설정했습니다.

## 주요 기능 및 개선 사항
본 프로젝트에서 진행된 유지보수 중 몇가지를 뽑아 소개합니다.

1. **사용자 계정 및 보안 관리 강화**   
사용자 계정의 생성, 관리 및 보안을 강화하여 시스템의 신뢰성을 높였습니다.

    - 개선 내용:
        - 안전한 회원가입 및 로그인 절차: ID 중복 확인, 8자 이상 비밀번호 길이 검증, 입력 비밀번호 해시화 저장 기능을 구현하여 사용자 정보의 보안을 강화했습니다.   
        - 유연한 계정 관리: 비밀번호 수정 및 회원 탈퇴 기능을 제공하며, 회원 탈퇴 시 해당 계정의 메일함을 자동으로 초기화하도록 하여 데이터 관리를 효율화했습니다.   
        - 접근 제어: 접속 유저의 권한에 따라 접근 가능한 페이지를 제한하는 기능을 추가하여 시스템의 보안 수준을 높였습니다.    
        ![image](https://github.com/user-attachments/assets/e289ba28-cf6f-4930-97ce-bf6e224928ec)

2. **메일 송수신 및 관리 기능 최적화**   
메일의 전송, 조회, 검색 및 첨부파일 관리를 개선하여 사용자 편의성과 시스템 효율성을 높였습니다.

    - 개선 내용:
        - 효율적인 메일 목록 조회: 전송한 메일 목록 및 내게 보낸 메일 목록을 확인할 수 있으며, 메일을 10개씩 나누어 페이지별로 보여주는 기능을 구현하여 대량의 메일도 쉽게 탐색할 수 있도록 했습니다.   
        - 강력한 메일 검색: 메일 제목 또는 보낸 ID로 메일을 검색할 수 있는 기능을 제공하여 사용자가 원하는 메일을 빠르게 찾을 수 있도록 지원합니다.   
        - 첨부파일 관리: 메일 삭제 시 서버에 저장된 첨부파일도 함께 삭제되도록 하여 불필요한 데이터 축적을 방지하고, 첨부파일을 download/(유저이름) 폴더 아래에 저장하도록 관리합니다.   
        - 편의 기능: 사용자 자신에게 메일을 보낼 수 있는 '나에게 메일 보내기' 기능을 제공합니다.   

3. **주소록 및 쓰레기통 기능 구현**   
사용자의 연락처 관리와 삭제된 메일의 효율적인 처리를 위한 기능을 추가 및 개선했습니다.

    - 개선 내용:
        - 주소록 관리: 사용자는 자신의 주소록을 조회, 추가, 수정, 삭제할 수 있으며, 주소록에 있는 주소를 사용하여 메일을 보낼 수 있도록 연동 기능을 제공합니다.   
        - 쓰레기통 기능: 사용자는 메일을 쓰레기통으로 보낼 수 있도록 하고, 쓰레기통 속의 메일을 완전히 삭제하거나 복원할 수 있는 기능을 구현하여 실수로 삭제된 메일에 대한 복구 가능성을 제공합니다.      

4. **코드 품질 및 잠재적 문제 예방**   
정적 분석 도구를 활용하여 코드 품질을 향상시키고 미래에 발생할 수 있는 잠재적 문제들을 사전에 식별했습니다.

    - 개선 내용:
        - SpotBugs 정적 분석: SpotBugs 정적 분석도구를 통해서 Bad practice(나쁜 습관), Dodgy code(잠재적 버그), Malicious code vulnerability(악성 코드 취약성), Internationalization(국제화) 등 현재에는 문제가 되지 않지만 미래에 문제가 발생할 수 있는 부분들을 파악하였습니다.

## **기술 스택**: 
기술 스택
본 프로젝트는 James Web Mail Server의 유지보수를 위해 다음과 같은 기술 스택을 활용했습니다.

    - 정정 분석 도구: SpotBugs
